## ファイルエクスプローラー

### ファイル名の拡張子を表示するか？

現状は拡張子を表示しています。
実際には拡張子は必須。

見た目をすっきりさせるには拡張子を非表示にした方が良い。
内部的にはドキュメントタイプを管理しているので、拡張子を表示しなくても問題ない。

名前変更等で拡張子を変更する必要が出てくる可能性もある。

ファイルエクスプローラー以外の人物一覧等のリスト表示部では拡張子を表示しないようにしても問題はなさそう。


## 複数ファイルに対して処理を行うパネル

互換的な処理の解消のために、拡張子の一括変換機能はどこかで欲しい。
なお、暫定的には`.novelaidattributes`で`.md`ファイルを`novel`扱いすることはできます。
Obsidian等と併用している場合にはこの機能で対応できます。
しかし、このソフトが実用的なレベルになってきたので、一括で`.md`から`.txt`に変換したい。
```pwsh
Get-ChildItem -Recurse -Filter *.md | Rename-Item -NewName { $_.FullName -replace '\.md$', '.txt' }
Get-ChildItem -Recurse -Filter *.md | Rename-Item -NewName { $_.FullName -replace '\.md$', '.txt' } -WhatIf
```
手動で処理する場合、上記コマンドをターミナル(`pwsh`)で実行すれば良い。
`-WatIf`付きの方は実際には処理をしない。
なお、同名の`.txt`ファイルが存在する場合はエラーになるので、エラー後に`-WatIf`で残っているファイルを確認すれば良い。

拡張子の変換は上記のように簡単にできはするが、`.md`のプロットファイルと混在している場合等に対処する場合、このソフトで`novel`な`.md`だけ処理したほうが良くはなるが、GUIで登録したファイルを一括変換で良いのでは？
で、これを実装するために複数ファイルを登録するGUIを用意する場合、その他の複数ファイルに対する処理も行えそうなので検討。

複数ファイルをまとめて出力する処理。
タイトルを含めるとか含めないとか選べると良いかも。
NotebookLMにファイル数を減らして入れたい場合とか、エピソードタイトルを挿入したい場合がある。
メタデータと組み合わせればできそう。
